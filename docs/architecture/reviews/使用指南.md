# 使用指南

## 文档目的

这份文档面向项目使用者和集成方，说明阶段 1 底座如何启动、配置、调用和排障。

## 运行前准备

### 1. Python 环境

建议版本：
- `Python 3.12`

项目当前虚拟环境：
- `.venv`

### 2. 安装依赖

如果 `.venv` 已经准备好，直接使用即可。

如需重新安装：

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### 3. 基础配置

建议最少配置：

```env
LLM_PROVIDER=mock
LOG_ENABLED=true
LOG_LEVEL=DEBUG
APP_DOWNLOAD_DIR=/app/download
```

说明：
- `APP_DOWNLOAD_DIR` 是上传文件、本地 OCR、本地 ASR、本地视频处理默认目录
- 不配置时默认值为 `/app/download`

### 4. 本地工具配置

如需接本地工具，可配置：

```env
OCR_TOOL_PATH=
ASR_TOOL_PATH=
VIDEO_PROBE_TOOL_PATH=
VIDEO_FRAME_TOOL_PATH=
VIDEO_AUDIO_TOOL_PATH=
```

说明：
- 不配置时会尝试从 `PATH` 自动发现

## CLI 使用方式

启动：

```powershell
.\.venv\Scripts\python.exe run.py
```

退出命令：
- `exit`
- `quit`
- `q`

### CLI 文本示例

```text
帮我写一句简短的自我介绍
```

### CLI 多模态命令式输入示例

图片：

```text
/image-file D:\demo\image.png | 请提取图片关键信息
```

音频：

```text
/audio-file D:\demo\audio.wav | 请转写并总结这个音频
```

视频：

```text
/video-file D:\demo\video.mp4 | 请总结这个视频
```

文件：

```text
/file-path D:\demo\doc.pdf | 请总结这个 PDF
```

## HTTP API 使用方式

启动 API：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.presentation.api:create_app --factory --host 127.0.0.1 --port 8011
```

### 健康检查

```powershell
Invoke-RestMethod http://127.0.0.1:8011/health
```

### 文本对话

```powershell
$body = @{
  message = "帮我写一句简短的自我介绍"
  user_name = "api-user"
  session_title = "API Session"
} | ConvertTo-Json

Invoke-RestMethod http://127.0.0.1:8011/chat -Method Post -ContentType "application/json" -Body $body
```

### 上传文件

```powershell
curl -X POST "http://127.0.0.1:8011/assets/upload" `
  -F "file=@D:\demo\image.png" `
  -F "kind=image" `
  -F "prompt=请提取图片里的关键信息" `
  -F "run_tools=true"
```

上传返回中会包含：
- `session_id`
- `turn_id`
- `task_id`
- `trace_id`
- `saved_path`
- `input_assets`
- `tool_results`

### 查询会话

```powershell
Invoke-RestMethod http://127.0.0.1:8011/sessions
Invoke-RestMethod http://127.0.0.1:8011/sessions/<session_id>
```

### 查询会话资产

```powershell
Invoke-RestMethod http://127.0.0.1:8011/sessions/<session_id>/assets
```

### 查询单个资产

```powershell
Invoke-RestMethod http://127.0.0.1:8011/assets/<asset_id>
```

### 查询任务

```powershell
Invoke-RestMethod http://127.0.0.1:8011/tasks
Invoke-RestMethod http://127.0.0.1:8011/tasks/<task_id>
```

### 查询工具

```powershell
Invoke-RestMethod http://127.0.0.1:8011/tools
```

### 直接执行工具

```powershell
$body = @{
  parameters = @{
    message = "hello tool"
  }
} | ConvertTo-Json

Invoke-RestMethod http://127.0.0.1:8011/tools/python_echo/execute -Method Post -ContentType "application/json" -Body $body
```

## 默认上传目录与覆盖方式

默认目录：

```text
/app/download
```

自定义覆盖：

```powershell
$env:APP_DOWNLOAD_DIR = "D:\agent_uploads"
```

然后再启动 CLI 或 API。

## 常见排障方法

### 1. 看任务

优先查：
- `GET /tasks`
- `GET /tasks/{task_id}`

### 2. 看资产

优先查：
- `GET /sessions/{session_id}/assets`
- `GET /assets/{asset_id}`

### 3. 看工具

优先查：
- `GET /tools`
- `tool_results`

### 4. 看日志

确保：

```env
LOG_ENABLED=true
LOG_LEVEL=DEBUG
```
