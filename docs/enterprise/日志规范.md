# 日志规范

## 1. 目的

统一日志记录方式，支撑排障和审计。

## 2. 基本要求

- 日志必须可开关
- 日志必须可配置级别
- 日志必须包含 trace 关联能力

## 3. 推荐字段

- `trace_id`
- `session_id`
- `turn_id`
- `task_id`
- `tool_name`
- `status`
- `error_code`

## 4. 推荐级别

- `DEBUG`
- `INFO`
- `WARNING`
- `ERROR`
- `CRITICAL`

## 5. 当前实现要求

- 进入 workflow 节点时记录
- 工具执行前后记录
- API 异常记录
- 失败任务记录
