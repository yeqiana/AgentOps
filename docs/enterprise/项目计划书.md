# 项目计划书

## 1. 项目名称

`AgentOps` Agent 底座项目

## 2. 项目目标

建设一个可扩展、可治理、可排障的 Agent 平台底座。

## 3. 当前范围

### 阶段 1

- 对话
- 上传
- 多模态输入
- 工具网关
- 任务追踪
- 数据落库

### 阶段 2

- 统一鉴权
- 限流与幂等
- trace service 基础能力
- 恢复治理
- 运行时配置中心
- 多 Agent 最小编排
- 正式角色协议
- 可切换执行协议

## 4. 当前收益

- 已形成可运行的 Agent 后端底座
- 已具备治理底座最小闭环
- 已具备多 Agent 最小编排能力
- 已为后续管理后台、异步任务、可视化观测打下基础

## 5. 当前阶段结果

- 阶段 1：已完成
- 阶段 2：进行中，当前进度约 `75% - 80%`

## 6. 阶段 2 已完成范围

- 统一鉴权
- 限流与幂等
- trace 查询与 trace-告警关联查询
- 恢复告警记录、查询和恢复策略配置化
- 模型与工具的重试、熔断和降级
- 工具白名单与上传安全限制
- 数据库驱动运行时配置中心
- 数据库驱动角色注册
- 正式角色协议
- 可切换执行协议
- 最小多 Agent 编排

## 7. 下一阶段方向

### 阶段 2 剩余重点

- 权限控制深化
- 策略中心深化
- 异步任务体系预留落地
- 多 Agent 协议继续扩展
- trace / alert 聚合能力增强

### 阶段 3 方向

- 可视化 trace
- 观测与监控面板
- 成本与配额治理
- 模型路由与策略引擎
- 异步任务平台正式化

## 8. 计划校正结论

二阶段原计划中的多项“规划中”能力已经提前落地，因此当前项目状态不应再描述为“治理和多 Agent 尚未开始”，而应描述为：

> 治理底座和最小多 Agent 编排已完成前半段，后续重点转向权限深化、策略深化和平台化能力扩展。
