# 项目计划书

## 1. 项目名称

`simple-ai-agent` Agent 底座项目

## 2. 项目目标

建设一个可扩展的分析型 Agent 后端底座。

## 3. 当前范围

阶段 1：
- 对话
- 上传
- 多模态输入
- 工具网关
- 任务追踪
- 数据落库

## 4. 预期收益

- 形成后续 Agent 应用的基础平台
- 降低后续 Web、插件、自动化应用的接入成本
- 为阶段 2 的多 Agent 编排打底

## 5. 当前阶段结果

阶段 1 已完成并可结项。

## 6. 下一阶段方向

- 多 Agent
- Router / Review
- 后台管理
- 更强治理能力

## 7. 后续治理能力补充

为满足企业级底座要求，后续规划明确补充以下能力：

### 阶段 2

- 统一鉴权
- 失败重试与恢复策略
- trace service
- 配置与版本治理
- 安全边界与工具策略
- 限流、幂等与配额控制

### 阶段 3

- 可视化 trace
- 观测与监控面板
- 更完整的权限治理体系
- 成本统计与预算治理
- 模型路由与策略引擎
- 评测与回归平台

## 8. 规划校正结论

为避免阶段顺序失衡，后续建设遵循以下原则：

- 先治理，后复杂编排
- 先统一入口和策略控制，后开放更多外部接入
- 先 trace 与重试，后多 Agent 和大规模自动化
- 先版本与配置治理，后大规模 Prompt 和 Provider 扩展
