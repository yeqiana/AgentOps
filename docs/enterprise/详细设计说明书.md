# 详细设计说明书

## 1. 设计范围

本说明书描述阶段 1 代码级设计和主要模块协作方式。

## 2. 主要模块

### 2.1 Presentation

职责：
- CLI 入口
- FastAPI 接口
- HTTP 请求参数解析

主要文件：
- `app/presentation/cli.py`
- `app/presentation/api/app.py`
- `app/presentation/api/schemas.py`

### 2.2 Application

职责：
- 会话初始化
- 输入解析
- 上传文件标准化
- Prompt 构建
- Session / Task 编排

主要文件：
- `app/application/agent_service.py`
- `app/application/upload_service.py`
- `app/application/prompt_builder.py`
- `app/application/services/session_service.py`
- `app/application/services/task_service.py`

### 2.3 Domain

职责：
- 状态模型
- 资产模型
- 错误模型

主要文件：
- `app/domain/models.py`
- `app/domain/errors.py`

### 2.4 Workflow

职责：
- Tool 节点
- Plan 节点
- Answer 节点

主要文件：
- `app/workflow/graph.py`
- `app/workflow/nodes.py`

### 2.5 Infrastructure

职责：
- LLM 调用
- 多媒体解析
- 上传存储
- 数据库
- 工具网关
- 日志

## 3. 关键设计点

### 3.1 统一状态

统一状态对象中包含：
- 用户输入
- 对话消息
- 输入资产
- 工具结果
- 执行标识
- 规划结果
- 最终回答

### 3.2 上传链路

上传流程：
1. 接收 `multipart/form-data`
2. 文件写入 `APP_DOWNLOAD_DIR`
3. 生成标准化 `InputAsset`
4. 可选触发 `tool_node`
5. 资产与任务落库

### 3.3 视频后处理链

当前视频链包括：
1. `video_ffprobe`
2. `video_ffmpeg_frame`
3. 关键帧回灌为图片资产
4. 关键帧 OCR
5. `video_ffmpeg_audio`
6. 音轨 ASR

### 3.4 PDF 解析链

PDF 解析使用：
- `pypdf`

处理逻辑：
- 尝试提取页面文本
- 提取失败时明确标记失败原因

## 4. 错误处理设计

- 参数错误
- 解析错误
- 模型错误
- 工具错误
- 持久化错误

所有错误通过 API 统一结构输出。
