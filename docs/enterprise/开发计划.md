# 开发计划

## 1. 阶段状态

- 阶段 1：已完成并结项
- 阶段 2：已完成开发，当前处于结项复核状态
- 阶段 3：规划中

## 2. 阶段 2 目标

阶段 2 的目标不是继续补基础问答能力，而是把阶段 1 的分析型底座推进为具备治理、可编排、可排障、可扩展的企业级 Agent 运行时最小版。

阶段 2 核心目标：
- 建立治理底座
- 建立最小多 Agent 编排
- 建立运行时配置中心
- 建立恢复与告警能力
- 建立正式角色协议与可切换执行协议
- 建立请求路由中台最小版
- 建立流式输出与异步任务深化
