# 开发计划

## 1. 阶段状态

- 阶段 1：已完成并结项
- 阶段 2：开发中，当前进度约 `85%`
- 阶段 3：规划中

## 2. 阶段 2 目标

阶段 2 的目标不再是补基础功能，而是把阶段 1 的分析型底座推进为可治理、可扩展、可排障的企业级 Agent 平台底座最小版。

核心目标：
- 建立治理底座
- 建立最小多 Agent 编排
- 建立运行时配置中心
- 建立恢复与告警能力
- 建立正式角色协议与可切换执行协议
- 为后续异步任务和可视化观测预留接口

## 3. 阶段 2 当前真实完成情况

### 3.1 已完成

- 统一鉴权
  - API Key
  - Bearer Token
- 最小 RBAC 授权
  - 角色定义
  - 权限定义
  - 主体角色分配
  - 高风险接口权限校验
- 基础治理能力
  - 限流
  - 幂等
- 失败恢复基础能力
  - LLM 重试
  - 工具重试
  - LLM 熔断
  - 工具熔断
  - LLM 降级到 `mock`
  - 工具 `soft-fail`
- trace service 基础能力
  - trace 落库
  - trace 查询
  - trace 与 task 关联
  - trace 与 alert 关联查询
- 告警能力
  - 告警落库
  - 告警查询
- 运行时配置中心
  - `sys_runtime_config`
  - `/config/runtime`
  - 数据库覆盖环境变量
- 安全边界控制
  - 工具白名单
  - 上传大小限制
  - 上传类型限制
- 最小多 Agent 编排
  - `router`
  - `debate`
  - `arbitration`
  - `critic`
  - `review`
- 正式角色协议
  - `support / challenge / planner / executor / arbitration / critic / reviewer`
- 数据库驱动角色注册
  - `sys_workflow_role`
  - `/workflow/roles`
- 可切换执行协议
  - `delegated`
  - `standard`
  - 执行协议已落库、已透出 API

### 3.2 部分完成

- 请求路由中台
  - 已有 workflow 级路由能力
  - 尚未形成独立“路由中台”
- 权限控制
  - 已有统一鉴权
  - 已有最小 RBAC
  - 尚未形成完整资源级授权、工具级细粒度授权和多租户权限模型
- 配置与版本治理
  - 已有运行时配置中心
  - 尚未形成 Prompt 版本、API 版本、变更审计完整体系
- 异步任务体系预留
  - 已有任务模型和任务查询接口
  - 尚未落地队列、worker、异步提交流程

### 3.3 未完成

- 可视化 trace
- 观测与监控面板
- 成本与配额治理
- 模型路由与策略引擎
- 完整异步任务平台
- 完整权限治理体系
