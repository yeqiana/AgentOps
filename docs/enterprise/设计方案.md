# 设计方案

## 1. 方案目标

为项目建立一个可以长期演进的 Agent 底座，而不是单点聊天脚本。

## 2. 方案选择

### 2.1 分层方案

选择：
- 五层结构

原因：
- 比单文件脚本更可维护
- 比过度拆分更适合当前规模

### 2.2 工作流方案

选择：
- LangGraph

原因：
- 适合状态流转
- 适合后续扩展 router / review / multi-agent

### 2.3 持久化方案

选择：
- 阶段 1 使用 SQLite

原因：
- 快速落地
- 便于测试

### 2.4 工具方案

选择：
- 本地工具注册中心 + 命令适配器

原因：
- 支持 OCR / ASR / 视频后处理
- 模型能力不足时可补强

## 3. 当前方案收益

- 稳定执行
- 可排障
- 易扩展

## 4. 后续阶段建议

- 阶段 2 做 router / review / multi-agent
- 阶段 3 做平台化和权限治理
