# 组件设计说明

## 1. 目的

说明当前核心组件及其职责边界。

## 2. 组件清单

### 2.1 CLI 组件

职责：
- 持续对话入口
- 输入读取
- 输出展示

### 2.2 API 组件

职责：
- HTTP 请求接入
- 上传文件接入
- 结构化响应输出

### 2.3 Upload Service 组件

职责：
- 上传文件落盘
- 推断资产类型
- 生成标准化资产

### 2.4 Workflow 组件

职责：
- 工具前处理
- 规划
- 回答

说明：
- 当前保持单 Agent 线性执行
- 后续多 Agent 编排将优先在该组件外围增加路由与协同层，而不是直接破坏现有主链

### 2.5 Tool Registry 组件

职责：
- 工具发现
- 工具注册
- 工具执行

### 2.6 Persistence 组件

职责：
- 会话落库
- 资产落库
- 任务落库
- 工具结果落库

说明：
- 当前已具备分段持久化能力
- 后续异步任务与多 Agent 场景将在该组件基础上扩展任务片段和执行记录

### 2.7 Request Route 预留组件

职责：
- 在入口层与工作流层之间执行统一请求分发
- 输出可追踪的路由决策结果

当前状态：
- 未独立实现
- 当前能力由 Workflow 内部最小工具路由替代

### 2.8 Auth / Policy 预留组件

职责：
- 身份认证
- 权限校验
- 工具授权
- 审计策略

当前状态：
- 未实现

### 2.9 Async Task 预留组件

职责：
- 任务排队
- 后台执行
- 状态流转
- 重试与取消

当前状态：
- 未实现

### 2.10 Media Loader 组件

职责：
- 图片解析
- 音频解析
- 视频解析
- 文件解析

## 3. 组件交互原则

- 表现层不直连底层命令
- workflow 不直接处理上传文件流
- persistence 不负责工具调度
- tool registry 不负责业务编排
- 请求路由、权限控制、异步任务应与主流程解耦扩展
