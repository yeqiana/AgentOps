# 组件设计说明

## 1. 目的

说明当前核心组件及其职责边界。

## 2. 组件清单

### 2.1 CLI 组件

职责：
- 持续对话入口
- 输入读取
- 输出展示

### 2.2 API 组件

职责：
- HTTP 请求接入
- 上传文件接入
- 结构化响应输出

### 2.3 Upload Service 组件

职责：
- 上传文件落盘
- 推断资产类型
- 生成标准化资产

### 2.4 Workflow 组件

职责：
- 工具前处理
- 规划
- 回答

### 2.5 Tool Registry 组件

职责：
- 工具发现
- 工具注册
- 工具执行

### 2.6 Persistence 组件

职责：
- 会话落库
- 资产落库
- 任务落库
- 工具结果落库

### 2.7 Media Loader 组件

职责：
- 图片解析
- 音频解析
- 视频解析
- 文件解析

## 3. 组件交互原则

- 表现层不直连底层命令
- workflow 不直接处理上传文件流
- persistence 不负责工具调度
- tool registry 不负责业务编排
