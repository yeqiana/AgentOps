# 数据库设计说明书

## 1. 说明

当前阶段 1 数据库使用 SQLite，目标是支撑最小闭环的会话、任务、资产和工具追踪。

## 2. 数据库选型

当前选型：
- SQLite

原因：
- 部署简单
- 便于阶段 1 快速收口
- 满足本地和单机服务验证

后续建议：
- PostgreSQL

## 3. 表设计

### 3.1 `users`

用途：
- 存储用户标识

主要字段：
- `id`
- `user_name`
- `created_at`

### 3.2 `sessions`

用途：
- 存储会话

主要字段：
- `id`
- `user_id`
- `title`
- `last_trace_id`
- `created_at`
- `updated_at`

### 3.3 `messages`

用途：
- 存储消息历史

主要字段：
- `id`
- `session_id`
- `turn_id`
- `trace_id`
- `role`
- `content`
- `created_at`

### 3.4 `assets`

用途：
- 存储输入资产

主要字段：
- `id`
- `session_id`
- `turn_id`
- `trace_id`
- `kind`
- `name`
- `source`
- `content`
- `storage_mode`
- `locator`
- `mime_type`
- `created_at`

### 3.5 `tasks`

用途：
- 存储任务状态

主要字段：
- `id`
- `session_id`
- `turn_id`
- `trace_id`
- `status`
- `user_input`
- `plan`
- `answer`
- `tool_count`
- `error_message`
- `created_at`
- `updated_at`

### 3.6 `tool_results`

用途：
- 存储工具执行明细

主要字段：
- `id`
- `task_id`
- `session_id`
- `turn_id`
- `trace_id`
- `tool_name`
- `success`
- `exit_code`
- `stdout`
- `stderr`
- `created_at`

## 4. 关系说明

- `users` 1 -> N `sessions`
- `sessions` 1 -> N `messages`
- `sessions` 1 -> N `assets`
- `sessions` 1 -> N `tasks`
- `tasks` 1 -> N `tool_results`

## 5. 查询场景

当前重点查询：
- 按 session 查消息
- 按 session 查资产
- 按 session 查任务
- 按 task 查工具结果
- 按 asset_id 查单资产

## 6. 后续演进建议

- 增加 `memories`
- 增加 `task_runs`
- 增加 `tool_calls`
- 增加索引和分区方案
