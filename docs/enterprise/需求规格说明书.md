# 需求规格说明书

## 1. 目标

定义阶段 1 底座必须满足的业务与技术需求。

## 2. 业务需求

### 2.1 基础对话需求

- 系统应支持持续多轮对话
- 系统应保留上下文
- 系统应支持 CLI 和 HTTP API 两种入口

### 2.2 多模态需求

- 系统应支持图片输入
- 系统应支持音频输入
- 系统应支持视频输入
- 系统应支持文件输入
- 系统应支持上传型输入和命令式输入

### 2.3 工具需求

- 系统应支持本地工具注册
- 系统应支持 OCR
- 系统应支持 ASR
- 系统应支持视频探测
- 系统应支持视频抽帧
- 系统应支持视频抽音轨

### 2.4 排障需求

- 系统应生成 `session_id / turn_id / task_id / trace_id`
- 系统应记录失败任务
- 系统应记录工具结果
- 系统应支持会话、任务、资产查询

## 3. 非功能需求

### 3.1 可维护性

- 必须保留五层结构
- 大模型接入必须正式分包
- 多媒体解析必须正式分包

### 3.2 可扩展性

- 后续应能接更多模型提供商
- 后续应能接更多工具
- 后续应能扩展多 Agent 工作流

### 3.3 可测试性

- 核心链路必须有自动化测试
- 上传、工具、持久化、工作流应被覆盖

## 4. 阶段 1 验收要求

- CLI 可用
- HTTP API 可用
- 数据库可落库
- 工具可调用
- 多模态输入可解析
- 失败路径可查
- 自动化测试通过
