# 需求规格说明书

## 1. 目标

定义阶段 1 底座必须满足的业务与技术需求。

## 2. 业务需求

### 2.1 基础对话需求

- 系统应支持持续多轮对话
- 系统应保留上下文
- 系统应支持 CLI 和 HTTP API 两种入口

### 2.2 多模态需求

- 系统应支持图片输入
- 系统应支持音频输入
- 系统应支持视频输入
- 系统应支持文件输入
- 系统应支持上传型输入和命令式输入

### 2.3 工具需求

- 系统应支持本地工具注册
- 系统应支持 OCR
- 系统应支持 ASR
- 系统应支持视频探测
- 系统应支持视频抽帧
- 系统应支持视频抽音轨

### 2.4 排障需求

- 系统应生成 `session_id / turn_id / task_id / trace_id`
- 系统应记录失败任务
- 系统应记录工具结果
- 系统应支持会话、任务、资产查询

### 2.5 多工具调用与请求路由需求

- 系统应支持在主流程中按输入资产类型触发对应工具
- 系统应为后续独立请求路由中台预留扩展点
- 路由决策结果后续应可被记录、查询和复盘

### 2.6 异步任务需求

- 当前阶段系统应提供统一 `task_id` 作为任务跟踪标识
- 后续阶段系统应支持异步任务提交、执行、重试、取消和结果查询
- 同步任务与异步任务应复用统一任务模型和追踪标识

### 2.7 权限控制需求

- 后续阶段系统应支持统一鉴权入口
- 后续阶段系统应支持工具级授权控制
- 后续阶段系统应支持审计记录与敏感操作留痕

### 2.8 多 Agent 编排需求

- 后续阶段系统应支持由单 Agent 执行链向多 Agent 协同演进
- 多 Agent 编排应复用现有状态、任务和追踪模型
- 多 Agent 间的任务交接后续应支持可追踪和可回放

## 3. 非功能需求

### 3.1 可维护性

- 必须保留五层结构
- 大模型接入必须正式分包
- 多媒体解析必须正式分包

### 3.2 可扩展性

- 后续应能接更多模型提供商
- 后续应能接更多工具
- 后续应能增加独立请求路由层
- 后续应能增加异步任务执行层
- 后续应能增加统一权限控制层
- 后续应能扩展多 Agent 工作流

### 3.3 可测试性

- 核心链路必须有自动化测试
- 上传、工具、持久化、工作流应被覆盖
- 路由、异步任务、权限控制和多 Agent 编排落地后应分别具备单元与集成测试

## 4. 阶段 1 验收要求

- CLI 可用
- HTTP API 可用
- 数据库可落库
- 工具可调用
- 多模态输入可解析
- 失败路径可查
- 自动化测试通过

## 5. 当前实现边界说明

当前版本已经支持：
- 单 Agent 主流程执行
- 基于输入资产类型的最小工具路由
- 任务与工具结果分段持久化
- API 层统一异常处理

当前版本尚未支持：
- 独立请求路由中台
- 异步任务执行体系
- 统一权限控制
- 多 Agent 编排
