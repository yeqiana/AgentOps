# 总体设计

## 1. 建设目标

`AgentOps` 的目标是建设一个可治理、可编排、可排障的 Agent 后端底座，而不是单一聊天应用。

平台建设重点：
- 支持模型与工具协同执行
- 支持多模态输入
- 支持企业级治理能力
- 支持多 Agent 编排
- 支持运行时配置与后续平台化演进

## 2. 当前阶段划分

### 阶段 1

- 建设分析型底座
- 打通模型、工具、多模态、持久化、日志和任务查询闭环

### 阶段 2

- 建设治理底座和最小多 Agent 编排
- 当前进度约 `88%`

### 阶段 3

- 建设可视化观测、监控、成本治理和平台级能力

## 3. 当前总体架构

系统采用五层结构：
- `presentation`
- `application`
- `domain`
- `workflow`
- `infrastructure`

总体原则：
- 接入层只负责协议
- 应用层只负责用例编排
- 领域层只负责稳定模型
- 工作流层只负责 Agent 编排
- 基础设施层只负责技术实现

## 4. 阶段 2 已落地能力

- 统一鉴权
- 最小 RBAC
- 限流与幂等
- trace service
- 告警记录与查询
- 重试、熔断、降级、恢复策略
- 数据库驱动运行时配置中心
- 数据库驱动角色注册
- 请求路由中台最小版
- 最小多 Agent 编排
- 正式角色协议
- 可切换执行协议
- CLI 与 SSE 流式对话
- 异步任务提交预留与后台执行器

## 5. 二阶段剩余重点

- 权限治理深化
- 策略中心深化
- 异步任务体系深化
- trace / alert 聚合能力继续增强

## 6. 后续演进方向

- 可视化 trace
- 观测与监控面板
- 成本与配额治理
- 模型路由与策略引擎
- 真正的异步任务平台
