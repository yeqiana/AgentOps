# 总体设计

## 1. 项目目标

`AgentOps` 的目标不是构建单一聊天应用，而是建设一个具备治理、编排、配置、排障和扩展能力的企业级 Agent 运行时底座。

## 2. 阶段划分

### 阶段 1

- 状态：已完成并结项
- 目标：完成分析型 Agent 底座闭环

### 阶段 2

- 状态：已完成并结项
- 目标：完成企业级 Agent 运行时最小版

### 阶段 3

- 状态：待启动
- 目标：推进可视化观测、平台化治理和完整异步任务平台

## 3. 总体架构

当前系统采用五层结构：
- `presentation`
- `application`
- `domain`
- `workflow`
- `infrastructure`

该结构已足以支撑阶段 1 和阶段 2 的交付目标。

## 4. 阶段 2 已落地能力

### 4.1 治理底座

- 统一鉴权
- 最小 RBAC
- 限流与幂等
- 重试、熔断、降级
- 恢复告警与统计

### 4.2 编排与执行

- 请求路由中台最小版
- 最小多 Agent 编排
- 正式角色协议
- 可切换执行协议

### 4.3 配置中心

- 数据库驱动运行时配置中心
- 配置变更审计与回查
- 运行时快照接口

### 4.4 交互与任务

- CLI 流式输出
- API SSE 流式对话
- 异步任务提交、运行、取消、重试
- 任务运行时快照与事件追踪

### 4.5 trace 与排障

- trace 统计、摘要、时间线、图谱
- alert 统计与关联查询
- 控制台聚合总览

## 5. 阶段 3 规划方向

- 可视化 trace
- 观测与监控面板
- 完整异步任务平台
- 完整权限治理体系
- 成本与配额治理
- 模型路由与策略引擎
