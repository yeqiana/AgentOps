# 培训材料

## 1. 培训目标

让开发、测试和接入方理解：
- 项目是什么
- 怎么跑
- 怎么查
- 怎么扩展

## 2. 培训对象

- 开发
- 测试
- 接口接入方
- 运维

## 3. 培训提纲

### 第一部分：项目定位

- Agent 底座是什么
- 阶段 1 做到了什么

### 第二部分：系统结构

- 五层结构
- tool_node / plan_node / answer_node

### 第三部分：使用方式

- CLI
- API
- 上传
- 查询

### 第四部分：排障方式

- trace_id
- task_id
- tool_results

### 第五部分：后续扩展

- 请求路由中台
- 异步任务体系
- 权限控制
- 多 Agent 编排

## 4. 培训重点补充

### 4.1 请求路由中台

培训口径：
- 当前仅支持基于输入资产类型的最小工具路由
- 后续将建设独立请求路由中台
- 路由中台用于统一任务分发、策略分发和多 Agent 路由入口

### 4.2 异步任务体系

培训口径：
- 当前仅支持同步执行主链
- 当前已具备 `task_id` 和任务查询基础
- 后续将扩展异步任务提交、排队、执行、重试和结果回查

### 4.3 权限控制

培训口径：
- 当前版本尚未实现统一鉴权与授权
- 后续将建设统一鉴权入口、资源级授权和工具级授权
- 权限控制会优先放在入口层和应用层，不散落在工作流节点中

### 4.4 多 Agent 编排

培训口径：
- 当前仅支持单 Agent 主流程
- 后续将扩展 Agent Router、角色 Agent 和结果汇聚机制
- 多 Agent 能力建立在统一状态、任务、路由和追踪模型基础上

## 5. 对外讲解注意事项

- 不要将请求路由中台表述为当前已交付能力
- 不要将异步任务体系表述为当前已上线能力
- 不要将权限控制表述为当前已落地能力
- 不要将多 Agent 编排表述为当前已上线能力
- 对外统一口径应为：当前已纳入正式设计和阶段规划，后续按计划建设
