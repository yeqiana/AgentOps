# 培训材料

## 1. 培训目标

本材料用于帮助新加入项目的开发、测试和运维人员快速理解 AgentOps 当前版本的结构、能力和使用方式。

培训目标如下：
- 理解项目定位
- 理解当前五层结构
- 理解主工作流
- 理解治理能力
- 理解当前阶段边界

## 2. 项目定位

AgentOps 是一个面向企业后端场景的 Agent 底座项目。

它当前已经具备：
- 对话能力
- 多模态输入能力
- 工具调用能力
- trace 与 alert 能力
- 恢复治理能力
- 最小多 Agent 编排能力

它当前尚未完整具备：
- 完整 RBAC
- 完整异步任务平台
- 可视化 trace 与监控面板

## 3. 五层结构速览

### 3.1 Presentation

作用：
- 接 CLI 和 HTTP 请求
- 做流式输出
- 做中间件治理

### 3.2 Application

作用：
- 做输入解析
- 组装上下文
- 构建 Prompt
- 编排工作流调用和持久化

### 3.3 Domain

作用：
- 定义模型
- 定义错误
- 定义仓储接口

### 3.4 Workflow

作用：
- 定义执行图
- 定义路由、协议和多角色节点

### 3.5 Infrastructure

作用：
- 负责模型、工具、数据库、追踪、告警、日志和存储

## 4. 当前主工作流

当前主链如下：

```text
tool_node -> router_node -> protocol_node -> plan_node
-> debate_node -> arbitration_node -> answer_node
-> critic_node -> review_node
```

说明：
- 并不是每次都会完整执行所有语义步骤
- `standard` 协议下会跳过部分多角色步骤

### 5.2 恢复

## 6. 常见理解误区

### 6.1 误区一：当前还是单 Agent 线性聊天

不准确。

当前已经有最小多 Agent 编排能力，只是还没有进入更复杂的多 Agent 平台阶段。

### 6.2 误区二：当前还没有鉴权

不准确。

当前已经有统一鉴权，但还没有完整 RBAC。

### 6.3 误区三：当前已经是完整平台

不准确。

当前仍处于阶段 2，中期重点是把治理和编排做深做稳。
