# AI Prompts

## 说明

本目录用于沉淀具有项目价值的用户提示词，作为长期可检索、可复用、可追溯的文档资产。

适用范围：
- 明确需求说明
- 架构约束
- 目录结构要求
- 页面或模块设计要求
- 与实现直接相关的任务提示词
- 对协作流程、代码规范、边界要求的补充说明

不建议归档：
- 单纯寒暄
- 无实质约束的简短确认
- 与项目实现无关的临时对话

## 目录建议

- `docs/ai-prompts/frontend/{YYYY-MM-DD}/`
- `docs/ai-prompts/backend/{YYYY-MM-DD}/`
- `docs/ai-prompts/architecture/{YYYY-MM-DD}/`
- `docs/ai-prompts/product/{YYYY-MM-DD}/`

## 文件命名建议

- 使用日期前缀
- 名称应具备可检索性
- 尽量使用 ASCII 文件名
- 避免 `prompt1.md` 这类无语义命名

推荐格式：
- `{YYYY-MM-DD}_{topic}.md`
- 如果同日多份，可追加序号：`{YYYY-MM-DD}_{topic}_01.md`

## 建议内容结构

每份提示词文档建议包含：
- 标题
- 日期
- 来源范围
- 主题标签
- 原始提示词
- 必要的上下文补充说明

## 去重规则

- 若同一提示词已归档，优先更新现有文档
- 若约束内容显著增强，可新增补充文档，或在原文档中增加补充要求区块
