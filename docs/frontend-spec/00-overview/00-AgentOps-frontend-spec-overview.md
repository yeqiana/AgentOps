# AgentOps 前端规范体系 v1

## 一、文档定位

本规范用于统一 AgentOps 多端产品的前端设计、交互、工程与落地方式。

适用范围包括：

- Web 管理后台
- 移动端管理
- AI 对话客户端
- 生成式创作平台

## 二、规范目标

1. 建立统一前端设计基线
2. 保证不同端的结构一致性与扩展性
3. 为 Codex / Cursor / 人工协作提供统一落地依据
4. 避免页面和代码随业务增长失控
5. 支持后续多端产品扩展而不推翻重来

## 三、规范分层

### L0：总纲层
定义整体原则、产品形态分类、适用范围与规范边界。

### L1：基础层
定义设计 token、状态系统、反馈系统、可访问性、文案规则等跨端共用能力。

### L2：工程层
定义前端项目结构、配置中心、组件开发规则、页面开发规则、API 与状态管理规范、AI 协作规范。

### L3：端形态层
按产品形态拆分：

- Web 管理后台
- Mobile 管理端
- AI 对话客户端
- 生成式创作平台

## 四、适用原则

- 通用规则优先进入基础层
- 端特有规则进入对应分册
- 页面规则必须从属于某一种产品形态
- 规范优先服务长期可维护，不追求短期炫技

## 五、一句话总结

> AgentOps 前端规范体系的目标是：统一设计语言、统一工程方式、统一落地标准，支撑多端持续扩展。
