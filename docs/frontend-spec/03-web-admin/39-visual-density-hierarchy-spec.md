# 39-Visual Density & Hierarchy Specification（视觉密度与层级规范）

---

## 一、目标

本规范用于统一 AgentOps Web Admin 控制台的：

- 信息密度（Density）
- 视觉层级（Hierarchy）
- 数据可扫读性（Scannability）

确保页面具备：

- 高信息承载能力
- 清晰主次结构
- 快速浏览与定位能力

---

## 二、设计原则

### 1. 控制台优先“信息密度”，而不是“呼吸感”

后台系统 ≠ 营销页面  
必须优先展示信息，而不是大面积留白。

---

### 2. 页面必须“可扫读”，而不是“可欣赏”

用户行为是：

- 快速浏览
- 定位异常
- 查找数据

而不是逐块阅读。

---

### 3. 所有 UI 必须表达“信息优先级”

界面必须让用户一眼知道：

- 哪些是核心数据
- 哪些是辅助信息
- 哪些可以忽略

---

## 三、页面密度规范（Density）

### 3.1 页面级（Page Level）

#### PageContainer

- 左右 padding：**16 ~ 24px**
- 页面最大宽度：限制（避免过宽）
- 页面顶部间距：收紧
- 区块间距：统一（16 / 20 / 24）

#### 禁止

- 大面积无意义留白
- 页面“拉伸填充空白”
- 单个模块占据过高空间但信息不足

---

### 3.2 区块级（Section Level）

#### Panel / Card

- 内边距：**12 ~ 16px**
- 区块间距：统一
- 标题区高度：收紧
- 不允许“空壳卡片”

---

### 3.3 组件级（Component Level）

| 组件 | 要求 |
|------|------|
| 表格行高 | 32 ~ 40px（紧凑） |
| 输入框高度 | 32 / 36px |
| 按钮高度 | 与输入框对齐 |
| 工具栏高度 | 统一 |

---

### 3.4 核心目标

> 单屏必须承载尽可能多的有效信息

---

### 4.1 标准层级结构

Header（系统层）
↓

Data（数据层）
### Density Mode（密度模式）

系统必须支持至少两种密度模式：

- compact（默认）
- comfortable

---

#### 规则

所有影响密度的 UI 必须通过 Token 控制：

- 页面 padding
- 卡片 padding
- 表格行高
- 区块间距
- 字号

---

#### 禁止

❌ 在组件中写死高度  
❌ 在页面中单独调整密度  
❌ 不通过 Token 控制间距  

---

#### 实现方式

通过 HTML root 控制：

```text
<html data-density="compact">
<html data-density="comfortable">
